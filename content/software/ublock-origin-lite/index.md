## Применение

Эта версия [uBlock Origin] предназначена для большинства браузеров на основе
[Chromium], которые насильно переходят на Manifest V3. Она использует
декларативный подход, что имеет как свои преимущества, так и недостатки. В
целом, Lite-версия менее эффективна, поэтому лучше использовать полноценный
[uBlock Origin], если в вашем браузере всё ещё есть такая возможность.

[uBlock Origin]: /software/ublock-origin
[Chromium]: /wiki/browser-engines#chromium--blink
[Manifest V3 API]: https://developer.chrome.com/docs/extensions/develop/migrate/what-is-mv3

### Преимущества

- Код выполняется браузером, а не расширением. *Теоретически*, это может быть
более безопасным и эффективным по ресурсам. Однако это
[ещё не было точно подтверждено](https://github.com/uBlockOrigin/uBOL-home/wiki/Frequently-asked-questions-(FAQ)#is-ubol-more-efficient-cpu--and-memory-wise-than-ubo).

- В Chromium-браузерах uBlock Origin Lite
[корректно фильтрует](https://github.com/uBlockOrigin/uBOL-home/wiki/Frequently-asked-questions-(FAQ)#what-are-the-upsides-of-an-mv3-based-content-blocker)
все сетевые запросы при запуске. Другие не полностью декларативные блокировщики,
включая uBlock Origin на Manifest V2, не могут этого достичь. Однако это не
является проблемой в браузерах на основе Firefox.

- В [базовом режиме фильтрации](#режим-фильтрации) uBlock Origin Lite не
запрашивает полный доступ к сайтам, хотя этот режим не установлен по умолчанию.

### Недостатки

- Обновления списков фильтров осуществляются вместе с расширением. Возможны
ситуации, когда на сайте происходит поломка, и обновление списка уже
опубликовано, но вам придётся подождать несколько дней пока выйдет обновление
uBlock Origin Lite вместе с новыми версиями списков фильтров.

- [Не все правила фильтрации удаётся адаптировать под Manifest V3](https://github.com/uBlockOrigin/uBOL-home/wiki/Frequently-asked-questions-(FAQ)#filtering-capabilities-which-cant-be-ported-to-mv3).
    - Повышенная вероятность [обнаружения](https://x.com/gorhill/status/1792688494999155147).
    - Невозможно фильтровать сложные параметры отслеживания регулярными
    выражениями.

## Установка

Вы можете установить uBlock Origin Lite напрямую из каталогов [Chrome] или
[Edge] (для Chromium-браузеров) и [AppStore] (для Safari). Для Firefox
расширение можно установить только из [GitHub].

[Chrome]: https://chromewebstore.google.com/detail/ddkjiahejlhfcafbddmgiahcphecmpfh
[Edge]: https://microsoftedge.microsoft.com/addons/detail/cimighlppcgcoapaliogpjjdehbnofhn
[AppStore]: https://apps.apple.com/app/id6745342698
[GitHub]: https://github.com/uBlockOrigin/uBOL-home/releases/latest

> [!important]
> Не забудьте разрешить работу в приватных окнах (режим Инкогнито) для
uBlock Origin Lite.
>
> [Не используйте](https://x.com/gorhill/status/1033706103782170625) другие
похожие расширения для блокировки контента вместе с uBlock Origin Lite: это
может потреблять больше ресурсов, снижать скорость загрузки страниц и нарушать
работу механизмов по предотвращению поломок сайтов и обнаружения.

## Конфигурация

В uBlock Origin Lite гораздо меньше функций и конфигурации, что менее технически
подкованным пользователям покажется удобным и понятным. Конфигурации по
умолчанию достаточно для защиты от вредоносного и навязчивого контента в
интернете без серьёзных поломок сайтов.

Вы можете сохранить вашу текущую конфигурацию в файл JSON.

### Режим фильтрации

В настройках можно установить режим фильтрации для всех сайтов по умолчанию:

- **Базовый** не требует разрешения на чтение и изменение данных на всех сайтах.
- **Оптимальный** *(по умолчанию)* требует разрешение на чтение и изменение
данных на всех сайтах.
- **Полный** может вызвать повышенное потребление ресурсов.

Вы можете переопределить режим фильтрации или отключить его для каждого сайта
отдельно в меню расширения.

Для большинства пользователей будет достаточно оставить оптимальный режим для
всех сайтов и изменять его для отдельных сайтов при необходимости.

### Списки фильтров

uBlock Origin Lite блокирует элементы согласно спискам фильтров, которые
поддерживаются волонтёрами. Обновления списков приходят вместе с новыми версиями
расширения. Вы не можете подписываться на внешние списки.

Рекомендации по спискам фильтров:

- Не активируйте слишком много фильтров: это может понизить производительность и
повысить вероятность поломки сайтов. Дублированные строки в списках убираются.

- **Используйте необходимые вам региональные фильтры.** Если вы посещаете
русскоязычные сайты, то убедитесь, что включён **RU AdList**. Дополнительно
можно включить фильтр **Counters** для блокировки счётчиков, параметров
отслеживания и прочего.

- Для очистки параметров отслеживания из ссылок включите
**AdGuard URL Tracking Protection**.

- Фильтры уведомлений о файлах куки лишь скрывают их, но не устанавливают
необходимые параметры для отключения сбора данных.

## Использование

### Убрать элемент

В меню расширения есть
[соответствующая функция](https://github.com/uBlockOrigin/uBOL-home/wiki/Quick-guide#remove-an-element)
со значком молнии. Убирает элементы с текущей страницы и пытается восстановить
возможность прокручивания (в случаях когда она была отключена). После
перезагрузки убранные элементы восстанавливаются. Это простой аналог удалению
элементов через инструменты разработчика.

### Создать свой фильтр

В меню расширения есть
[соответствующая функция](https://github.com/uBlockOrigin/uBOL-home/wiki/Quick-guide#create-a-custom-filter)
со значком перечёркнутого глаза. Позволяет создать фильтр для постоянного
скрытия элементов со страницы.

При нажатии на элемент с помощью этого инструмента откроется диалоговое окно,
позволяющее тонко настроить предложенный фильтр. Всегда следует проверять, что
создаваемый фильтр оптимально нацелен на элементы, которые вы хотите удалить.
Вы можете регулировать это с помощью ползунка. Если раскрыть окно, то появится
больше настроек.

![Создание своего фильтра в uBlock Origin Lite. Нужно выбрать элемент, затем
появляется диалоговое окно, где можно настроить выбор ползунком. Если раскрыть
окно, то появится более гибкий интерфейс для создания
фильтра.](create-custom-filter.png)

Созданные фильтры можно посмотреть во вкладке «Пользовательские фильтры» в
настройках расширения. Здесь их можно редактировать и удалять, экспортировать в
текстовый файл и импортировать.

> [!note]
> Это текстовое поле предназначено только для **косметических** фильтров, то
есть фильтров элементов страниц, а не адресов сайтов.

### Режим разработчика

Продвинутые пользователи, которые имеют представление о работе
[declarativeNetRequest API], могут включить [режим разработчика], чтобы получить
доступ к просмотру содержимого списков фильтров и созданию своего списка
во вкладке «Разработка».

Если выбрать «Пользовательские правила DNR», то можно вписать свои сетевые
фильтры в формате, похожем на YAML. Также можно вставить фильтры формата
Adblock, и они конвертируются автоматически.

Неправильное использование этих инструментов для разработчиков может нанести
вред вашей конфиденциальности и безопасности.

[declarativeNetRequest API]: https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/API/declarativeNetRequest
[режим разработчика]: https://github.com/uBlockOrigin/uBOL-home/wiki/Dashboard:-Develop

## Исправление проблем

Если сайт отображается некорректно или не работает вовсе, то это может быть
вызвано многими факторами, поэтому для начала попробуйте следующее:

- Отключите uBlock Origin Lite.
- Откройте страницу в другом браузере.
- Отключите дополнительные списки фильтров.

В случае некорректной работы фильтров uBlock Origin Lite (поломка сайта,
обнаружение, реклама, помехи) можно сообщить о проблеме. Волонтёры постараются
решить её как можно быстрее. Используйте соответствующую функцию в меню
расширения (значок чата). Для создания отчёта потребуется аккаунт на GitHub и
немного времени, чтобы убедиться в том, что нет дубликатов, и вы делаете
правильные действия.
